Current runtime behavior uses `POST /api/oauth/email/bind`.

This endpoint binds an email address to the current signed-in account and no longer accepts the legacy GET query form.

Request body:

```json
{
  "email": "user@example.com",
  "code": "123456"
}
```

Notes:

- A valid logged-in session is required.
- The verification code must be issued first via `/api/verification`.
- Do not use the old `GET /api/oauth/email/bind?email=...&code=...` form.
