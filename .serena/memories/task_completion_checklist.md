# 任务完成检查清单

## 完成任务后应该做什么

### 代码质量检查

#### 后端（Go）
- [ ] 运行 `go fmt ./...` 格式化代码
- [ ] 运行 `go vet ./...` 检查代码问题
- [ ] 运行 `go test ./...` 执行测试
- [ ] 确保所有测试通过
- [ ] 检查代码覆盖率（可选）：`go test -cover ./...`

#### 前端（React）
- [ ] 运行 `bun run lint` 检查代码格式
- [ ] 运行 `bun run lint:fix` 修复格式问题
- [ ] 运行 `bun run eslint` 检查 ESLint 问题
- [ ] 运行 `bun run eslint:fix` 修复 ESLint 问题
- [ ] 运行 `bun run build` 验证生产构建成功
- [ ] 检查构建输出中是否有警告

### 国际化检查（前端）
- [ ] 如果添加了新的用户界面文本，运行 `bun run i18n:extract` 提取翻译键
- [ ] 运行 `bun run i18n:lint` 检查翻译文件完整性
- [ ] 确保所有新文本都有中文翻译

### 数据库相关
- [ ] 如果修改了数据库模型，确保迁移脚本正确
- [ ] 测试所有三个数据库（SQLite、MySQL、PostgreSQL）
- [ ] 验证数据库兼容性（列引用、布尔值、SQL 函数等）

### Git 提交
- [ ] 运行 `git status` 检查未暂存的更改
- [ ] 运行 `git diff` 审查更改内容
- [ ] 暂存相关文件：`git add <files>`
- [ ] 创建有意义的提交信息：`git commit -m "type: description"`
- [ ] 推送到远程分支：`git push origin <branch-name>`

### 测试
- [ ] 手动测试新功能
- [ ] 测试边界情况和错误处理
- [ ] 验证与现有功能的兼容性
- [ ] 如果适用，编写自动化测试

### 文档
- [ ] 更新 README.md（如果需要）
- [ ] 添加代码注释（复杂逻辑）
- [ ] 更新 API 文档（如果修改了 API）
- [ ] 记录任何新的环境变量或配置

### 性能和安全
- [ ] 检查是否有 N+1 查询问题
- [ ] 验证输入验证和错误处理
- [ ] 检查是否有敏感信息泄露
- [ ] 验证速率限制和认证

### 构建验证
- [ ] 后端：`make build-backend` 成功
- [ ] 前端：`bun run build` 成功
- [ ] 完整构建：`make all` 成功（如果适用）

### 最终检查
- [ ] 代码审查（自我审查或请求他人审查）
- [ ] 所有测试通过
- [ ] 没有 linting 错误或警告
- [ ] 提交信息清晰有意义
- [ ] 功能按预期工作

## 常见问题排查

### 前端构建失败
- 检查 Node.js/Bun 版本
- 清除 node_modules：`rm -rf node_modules && bun install`
- 检查 Vite 配置
- 查看构建错误日志

### 后端编译失败
- 检查 Go 版本：`go version`（需要 1.26.2+）
- 检查依赖：`go mod tidy`
- 清除构建缓存：`go clean -cache`
- 查看编译错误信息

### 数据库连接问题
- 检查 .env 中的数据库配置
- 验证数据库服务是否运行
- 检查数据库凭证和权限
- 查看日志获取详细错误信息

### 测试失败
- 运行单个测试：`go test -run TestName ./...`
- 查看测试输出获取详细信息
- 检查测试数据和 mock 设置
- 验证测试环境配置
