# setting/AGENTS.md

`setting/` 管理系统、运营、模型、倍率、性能和控制台配置。

## 规则

- 配置通过现有 registry/config 模型注册和读取，避免散落全局变量写入。
- 配置值必须有明确默认值、合法值校验和错误信息。
- `theme.frontend` 属于双 UI 关键配置，只能是 `default` 或 `classic`；默认兼容路径是 classic。
- 修改配置结构时同步 controller、前端系统设置页面和持久化逻辑。
- 倍率、价格、状态码区间、限流等配置解析失败必须显式返回错误，不能回落到看似成功的默认值。

## 验证

- 改配置解析或校验后执行 `go test ./setting/...`。
- 影响前端系统设置时同时构建受影响 UI。
